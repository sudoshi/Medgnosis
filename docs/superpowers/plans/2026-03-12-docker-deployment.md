# Dockerized Deployment Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace systemd-based production deployment with Docker Compose, so deployment becomes `docker compose up --build -d` with no sudo required.

**Architecture:** Multi-stage Docker builds for API (node:22-alpine) and frontend (vite build → nginx:1.27-alpine). All app services (nginx, api, worker, solr, redis) on a `medgnosis` bridge network. PostgreSQL stays on the host, reached via `host.docker.internal`. nginx serves the SPA and reverse-proxies `/api/v1/` and `/health` to the API container.

**Tech Stack:** Docker, Docker Compose, nginx 1.27, node:22-alpine, Solr 9.7, Redis 7, BullMQ workers

**Spec:** `docs/superpowers/specs/2026-03-12-docker-tui-installer-design.md`

### Task 2: Create API Dockerfile

**Files:**
- Create: `docker/api/Dockerfile`

Multi-stage build: builder stage compiles TypeScript via turbo, runtime stage copies only `dist/` and production `node_modules`. The API and worker share the same image — the entrypoint determines which process runs.

**Key decisions:**
- `node:22-alpine` for small image size (~200MB final)
- `ENV API_PORT=3002` baked in as safety net (config.ts defaults to 3000)
- Turbo prune for minimal build context would be ideal but adds complexity — use full monorepo build instead since the image is internal-only
- Worker runs from same image with different `command:`

- [ ] **Step 1: Create docker/api/ directory**

Run: `mkdir -p docker/api`

- [ ] **Step 2: Write API Dockerfile**

```dockerfile
# =============================================================================
# Medgnosis API — Multi-stage Docker build
# Stage 1: Install deps + compile TypeScript
# Stage 2: Slim runtime with only dist/ + production node_modules
# =============================================================================

# --------------- Stage 1: Builder ---------------
FROM node:22-alpine AS builder

WORKDIR /app

# Copy workspace structure for dependency resolution
COPY package.json package-lock.json turbo.json tsconfig.base.json ./
COPY apps/api/package.json apps/api/tsconfig.json ./apps/api/
COPY packages/shared/package.json packages/shared/tsconfig.json ./packages/shared/
COPY packages/db/package.json packages/db/tsconfig.json ./packages/db/
COPY packages/solr/package.json packages/solr/tsconfig.json ./packages/solr/

# Install all dependencies (including devDependencies for build)
RUN npm ci

# Copy source code
COPY apps/api/src ./apps/api/src
COPY packages/shared/src ./packages/shared/src
COPY packages/db/src ./packages/db/src
COPY packages/db/migrations ./packages/db/migrations
COPY packages/db/seeds ./packages/db/seeds
COPY packages/solr/src ./packages/solr/src

# Build all packages in dependency order
RUN npx turbo build --filter=@medgnosis/api

# --------------- Stage 2: Runtime ---------------
FROM node:22-alpine AS runtime

WORKDIR /app

# Safety net: config.ts defaults to 3000, but nginx proxies to 3002
ENV API_PORT=3002
ENV NODE_ENV=production

# Copy package files for production install
COPY package.json package-lock.json ./
COPY apps/api/package.json ./apps/api/
COPY packages/shared/package.json ./packages/shared/
COPY packages/db/package.json ./packages/db/
COPY packages/solr/package.json ./packages/solr/

# Install production dependencies only
RUN npm ci --omit=dev --workspace=apps/api --workspace=packages/shared --workspace=packages/db --workspace=packages/solr \
    && npm cache clean --force

# Copy compiled output from builder
COPY --from=builder /app/apps/api/dist ./apps/api/dist
COPY --from=builder /app/packages/shared/dist ./packages/shared/dist
COPY --from=builder /app/packages/db/dist ./packages/db/dist
COPY --from=builder /app/packages/db/migrations ./packages/db/migrations
COPY --from=builder /app/packages/db/seeds ./packages/db/seeds
COPY --from=builder /app/packages/solr/dist ./packages/solr/dist

# Non-root user for security
RUN addgroup -S medgnosis && adduser -S medgnosis -G medgnosis
USER medgnosis

EXPOSE 3002

# Default: API server. Override with command: ["node", "apps/api/dist/worker.js"] for worker
CMD ["node", "apps/api/dist/server.js"]
```

**Files:**

The nginx container serves double duty: builds the Vite SPA in stage 1, then serves it as static files with reverse proxy rules in stage 2.

- [ ] **Step 1: Create docker/nginx/ directory**

Run: `mkdir -p docker/nginx`

- [ ] **Step 2: Write nginx default.conf**

```nginx
# =============================================================================
# Medgnosis — nginx reverse proxy + SPA server
# Sits between Apache (or direct) and the API container
# =============================================================================

upstream api {
    server api:3002;
}

server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    # ── Gzip compression ──
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml image/svg+xml;
    gzip_min_length 256;

    # ── Health check (for Docker healthcheck) ──
    location = /nginx-health {
        access_log off;
        return 200 'ok';
        add_header Content-Type text/plain;
    }

    # ── API health (no auth, proxied to Fastify) ──
    location = /health {
        proxy_pass http://api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # ── WebSocket endpoint ──
    location /api/v1/ws {
        proxy_pass http://api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400s;
    }

    # ── API routes ──
    location /api/v1/ {
        proxy_pass http://api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts for slow queries (large patient lists, AI insights)
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }

    # ── Swagger docs (proxied, no auth on Fastify side) ──
    location /docs {
        proxy_pass http://api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # ── Solr admin UI (optional, for debugging) ──
    location /solr/ {
        proxy_pass http://solr:8983;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # ── SPA catch-all: serve index.html for client-side routing ──
    location / {
        try_files $uri $uri/ /index.html;

        # Cache static assets aggressively
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }
}
```

### Task 4: Create docker-compose.yml

**Files:**
- Create: `docker-compose.yml`

This is the main orchestration file. It replaces both `docker-compose.demo.yml` (for app infra) and the systemd services (for the API). The demo file is NOT deleted — it's still useful for developers who want to run just the infrastructure without containerizing the app.

**Key decisions:**
- Worker runs from the same API image with a different command
- Solr always starts (API gracefully degrades when `SOLR_ENABLED=false`)
- `host.docker.internal` for host PostgreSQL access (Docker >= 20.10)
- Mailpit (not MailHog) on `dev` profile only
- All services get `restart: unless-stopped`

- [ ] **Step 1: Write docker-compose.yml**

```yaml
# =============================================================================
# Medgnosis — Production Docker Compose
# Replaces systemd-based deployment. PostgreSQL stays on host.
# Usage: docker compose up --build -d
# =============================================================================

name: medgnosis

services:
  # ── nginx: reverse proxy + SPA static files ──
  nginx:
    build:
      context: .
      dockerfile: docker/nginx/Dockerfile
    ports:
      - "${NGINX_PORT:-3081}:80"
    depends_on:
      api:
        condition: service_healthy
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "-q", "--spider", "http://localhost:80/nginx-health"]
      interval: 5s
      timeout: 3s
      retries: 5
      start_period: 10s
    deploy:
      resources:
        limits:
          memory: 256M
          cpus: "0.5"
    networks:
      - medgnosis

  # ── api: Fastify REST API ──
  api:
    build:
      context: .
      dockerfile: docker/api/Dockerfile
    env_file:
      - .env
    environment:
      # Docker-internal URLs (override any .env defaults for container networking)
      REDIS_URL: redis://redis:6379
      SOLR_URL: http://solr:8983/solr
    extra_hosts:
      - "host.docker.internal:host-gateway"
    depends_on:
      redis:
        condition: service_healthy
      solr:
        condition: service_healthy
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "node", "-e", "fetch('http://localhost:3002/health').then(r=>{if(!r.ok)throw 1}).catch(()=>process.exit(1))"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    deploy:
      resources:
        limits:
          memory: 1G
          cpus: "2"
    networks:
      - medgnosis

  # ── worker: BullMQ background job processor ──
  worker:
    build:
      context: .
      dockerfile: docker/api/Dockerfile
    command: ["node", "apps/api/dist/worker.js"]
    env_file:
      - .env
    environment:
      REDIS_URL: redis://redis:6379
    extra_hosts:
      - "host.docker.internal:host-gateway"
    depends_on:
      redis:
        condition: service_healthy
    restart: unless-stopped
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: "1"
    networks:
      - medgnosis

  # ── redis: cache, BullMQ queue, WebSocket pub/sub ──
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    ports:
      - "${REDIS_PORT:-6379}:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 10
      start_period: 5s
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: "1"
    networks:
      - medgnosis

  # ── solr: search + clinical cores ──
  solr:
    image: solr:9.7-slim
    command: >
      bash -c "
        precreate-core search /opt/solr/server/solr/configsets/search;
        precreate-core clinical /opt/solr/server/solr/configsets/clinical;
        exec solr-foreground
      "
    ports:
      - "${SOLR_PORT:-8984}:8983"
    environment:
      SOLR_JAVA_MEM: "-Xms2g -Xmx4g"
    volumes:
      - solr_data:/var/solr
      - ./solr/search/conf:/opt/solr/server/solr/configsets/search/conf:ro
      - ./solr/clinical/conf:/opt/solr/server/solr/configsets/clinical/conf:ro
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-sf", "http://localhost:8983/solr/admin/info/system"]
      interval: 10s
      timeout: 5s
      retries: 10
      start_period: 30s
    deploy:
      resources:
        limits:
          memory: 4G
          cpus: "2"
    networks:
      - medgnosis

  # ── mailpit: dev email capture (replaces MailHog) ──
  mailpit:
    image: axllent/mailpit
    ports:
      - "1025:1025"
      - "8025:8025"
    restart: unless-stopped
    profiles:
      - dev
    networks:
      - medgnosis

volumes:
  redis_data:
  solr_data:

networks:
  medgnosis:
    driver: bridge
```

## Chunk 2: Deploy Script + Integration Testing

### Task 7: Create deploy script

**Files:**
- Create: `scripts/deploy.sh`

Replaces `scripts/deploy-production.sh` for Docker-based deployments. Supports flags for partial rebuilds, logs, and status.

- [ ] **Step 1: Write scripts/deploy.sh**

```bash
#!/usr/bin/env bash
# =============================================================================
# Medgnosis — Docker Deploy Script
# Usage:
#   ./scripts/deploy.sh              # Full rebuild + restart
#   ./scripts/deploy.sh --api        # Rebuild API only
#   ./scripts/deploy.sh --frontend   # Rebuild nginx (frontend) only
#   ./scripts/deploy.sh --restart    # Restart without rebuild
#   ./scripts/deploy.sh --logs       # Tail all container logs
#   ./scripts/deploy.sh --status     # Show service health
#   ./scripts/deploy.sh --down       # Stop all containers
# =============================================================================

set -euo pipefail

cd "$(dirname "$0")/.."

COMPOSE="docker compose"

# Colors
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
BLUE='\033[0;34m'
NC='\033[0m'

log()  { echo -e "${BLUE}[deploy]${NC} $1"; }
ok()   { echo -e "${GREEN}[  ok  ]${NC} $1"; }
warn() { echo -e "${YELLOW}[ warn ]${NC} $1"; }
err()  { echo -e "${RED}[error ]${NC} $1"; }

# ── Pre-flight checks ──
preflight() {
    if ! command -v docker &>/dev/null; then
        err "Docker is not installed. See https://docs.docker.com/engine/install/"
        exit 1
    fi

    if ! docker info &>/dev/null; then
        err "Docker daemon is not running."
        exit 1
    fi

    local docker_version
    docker_version=$(docker version --format '{{.Server.Version}}' 2>/dev/null || echo "0.0.0")
    local major minor
    major=$(echo "$docker_version" | cut -d. -f1)
    minor=$(echo "$docker_version" | cut -d. -f2)
    if (( major < 20 || (major == 20 && minor < 10) )); then
        err "Docker >= 20.10 required (found $docker_version). host.docker.internal needs this version."
        exit 1
    fi

    if [ ! -f .env ]; then
        warn ".env file not found. Copying from .env.example..."
        cp .env.example .env
        warn "Edit .env with your database credentials before deploying."
        exit 1
    fi
}

# ── Wait for healthchecks ──
wait_healthy() {
    local service=$1
    local max_wait=${2:-120}
    local waited=0
    local container="medgnosis-${service}-1"

    log "Waiting for $service to be healthy..."
    while [ $waited -lt $max_wait ]; do
        local health
        health=$(docker inspect --format='{{.State.Health.Status}}' "$container" 2>/dev/null || echo "unknown")

        if [ "$health" = "healthy" ]; then
            ok "$service is healthy"
            return 0
        fi
        sleep 2
        waited=$((waited + 2))
    done
    warn "$service did not become healthy within ${max_wait}s"
    return 1
}

# ── Status table ──
status() {
    log "Service status:"
    echo ""
    $COMPOSE ps --format "table {{.Name}}\t{{.Status}}\t{{.Ports}}"
    echo ""
}

# ── Main ──
case "${1:-}" in
    --api)
        preflight
        log "Rebuilding API..."
        $COMPOSE build api
        $COMPOSE up -d api worker
        wait_healthy api
        status
        ;;
    --frontend)
        preflight
        log "Rebuilding frontend..."
        $COMPOSE build nginx
        $COMPOSE up -d nginx
        wait_healthy nginx
        status
        ;;
    --restart)
        preflight
        log "Restarting all services..."
        $COMPOSE restart
        wait_healthy redis 30
        wait_healthy solr 60
        wait_healthy api
        wait_healthy nginx
        status
        ;;
    --logs)
        $COMPOSE logs -f --tail=100
        ;;
    --status)
        status
        ;;
    --down)
        log "Stopping all services..."
        $COMPOSE down
        ok "All services stopped."
        ;;
    --help|-h)
        echo "Usage: $0 [--api|--frontend|--restart|--logs|--status|--down|--help]"
        echo ""
        echo "  (no flag)    Full rebuild + restart all services"
        echo "  --api        Rebuild and restart API + worker only"
        echo "  --frontend   Rebuild and restart nginx (frontend) only"
        echo "  --restart    Restart all without rebuilding"
        echo "  --logs       Tail all container logs"
        echo "  --status     Show service health status"
        echo "  --down       Stop all containers"
        exit 0
        ;;
    "")
        preflight
        log "Full rebuild + deploy..."
        $COMPOSE build
        $COMPOSE up -d
        log "Waiting for services..."
        wait_healthy redis 30
        wait_healthy solr 60
        wait_healthy api
        wait_healthy nginx
        echo ""
        ok "Medgnosis is running!"
        status
        ;;
    *)
        err "Unknown flag: $1. Use --help for usage."
        exit 1
        ;;
esac
```

## Notes

### What this plan does NOT include

1. **TUI Installer** — separate plan, to be written after Docker setup is validated
2. **Solr core creation** — handled by the Solr `command:` entrypoint which runs `precreate-core` before starting Solr; existing `solr_data` volume preserves cores across restarts
3. **Database migrations inside Docker** — migrations run from host (`npm run db:migrate`) or via the TUI installer; the API container does NOT auto-run migrations on startup (this is intentional — migrations are a manual step)
4. **CI/CD changes** — out of scope per spec
5. **Deleting docker-compose.demo.yml** — kept for developers who run the app natively with Docker infra only

### Rollback

If Docker deployment fails:
```bash
docker compose down
sudo systemctl enable medgnosis-api medgnosis-worker
sudo systemctl start medgnosis-api medgnosis-worker
```
